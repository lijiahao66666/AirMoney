# AirMoney（哎呀钱）产品方案

## 一、产品定位

- **Slogan**：在意你的每一笔钱
- **核心理念**：买前有咨询（该不该买），买后有记账 + 分析反思（花得值不值）
- **品牌**：AirMoney 谐音「哎呀钱」，传达「在意钱」的态度

---

## 二、三个模块总览

| 模块 | 触发时机 | 核心能力 | 大模型作用 |
|------|----------|----------|------------|
| **记账** | 闹钟提醒 + 手动补充 | 记录每笔支出 | 不依赖 |
| **分析** | 单次记账后 / 选时间段 | 花得值不值、习惯与建议 | 核心 |
| **咨询** | 想买某样东西时 | 要不要买、买什么更值 | 核心 |

---

## 三、模块一：记账

### 3.1 功能

1. **闹钟提醒（默认）**
   - 每天 13:00、20:00 推送「今天有记账吗？」
   - 点击直接进入记账页
   - 用户可改时间、次数、是否关闭

2. **手动记账**
   - 金额、分类、备注、支付方式
   - 支持快捷分类（餐饮、交通、购物等）
   - 可补记历史日期

3. **记完后**
   - 保存后自动进入「分析反思」流程（见模块二）

### 3.2 闹钟配置

- 默认：13:00、20:00 各一次
- 可调：时间、次数（最多 3 次/天）、是否周末单独设置
- 文案示例：「哎呀钱！今天有花钱吗？来记一笔吧」

### 3.3 数据结构（简要）

```
Bill {
  id, amount, category, note, payMethod,
  date, createdAt
}
```

## 四、模块二：分析

### 4.1 两种分析场景

| 场景 | 触发 | 内容 |
|------|------|------|
| **单次分析** | 刚记完一笔 | 这笔钱花得值不值、怎么想 |
| **周期分析** | 用户选择时间范围 | 这段时间的消费习惯、可省之处 |

### 4.2 单次记账后分析（流程）

```
用户记完一笔 → 弹窗/跳转「分析」页 → 调大模型分析 → 展示结果
```

**传给大模型的内容**：
- 本次记录：金额、分类、备注、日期
- 可选：该分类近 7/30 天汇总

**大模型任务**：
1. 判断这笔是否偏冲动 / 不必要
2. 简短建议（50–100 字）
3. 1–2 个反思问题

**输出示例**：

```
这笔 38 元的奶茶支出——

分析：你今天已有 2 笔餐饮，这杯奶茶可能不是刚需。
建议：下次想喝前，先问自己「不喝会怎样？」。

反思：这 38 元，如果不花，你会用来做什么？
```

**交互**：
- 用户可点「记下反思」把回答保存
- 或跳过

### 4.3 周期分析（流程）

```
用户选「分析」→ 选择时间范围（本周/本月/自定义）→ 调大模型分析 → 展示结果
```

**传给大模型的内容**：
- 该时间段内所有账单（按日期、分类聚合）
- 分类占比、总额、最高几笔等统计

**大模型任务**：
1. 概括消费特点（高频类、增长趋势）
2. 指出可能的浪费点
3. 给 2–3 条可执行建议

**输出示例**：

```
本周支出 1,247 元，比上周多 18%

主要分布：
- 餐饮 42%（524 元）— 工作日外卖偏多
- 购物 28%（349 元）— 有 3 笔 100+ 的冲动消费
- 交通 15%（187 元）

建议：
1. 餐饮可尝试带餐 2–3 天，预计能省约 150 元
2. 大额购物前用「买前咨询」再决策
3. 你已有 5 笔「奶茶/咖啡」类，可尝试减少频率
```

**交互**：
- 用户可「记下反思」或「生成报告」保存
- 支持分享（截图或文案）

### 4.4 分析模块的系统 Prompt（示例）

```
你是「哎呀钱」的消费分析顾问。

输入：用户的账单数据（单笔或一段时间）

任务：
1. 单笔：判断是否必要、是否冲动，给 1–2 条建议和 1 个反思问题
2. 周期：概括消费结构、找浪费点，给 2–3 条可执行建议

要求：
- 语气温和、不指责
- 建议具体、可操作
- 控制字数：单笔约 100 字，周期约 200 字
- 用「建议」「可以尝试」等措辞
```

## 五、模块三：咨询

### 5.1 功能

用户在「想买」但还没买时，来咨询是否值得买、该买什么样的。

### 5.2 流程

```
用户输入「我想买 XXX」
    ↓
大模型先问 3–5 个问题（分批或一次）
    ↓
用户回答
    ↓
大模型给出结论 + 建议
```

### 5.3 咨询的提问逻辑（可固化在 Prompt）

| 问题类型 | 示例 |
|----------|------|
| 已有替代 | 你已经有类似的东西了吗？ |
| 用途 | 买来主要做什么用？ |
| 预算 | 大概打算花多少？ |
| 必要性 | 不买的话会怎样？ |
| 时机 | 是必须马上买，还是可以等等？ |

### 5.4 输出格式（示例）

```
结论：建议 / 可以买 / 建议不买 / 可以再等等

理由：（2–3 句）

如果要买：推荐价位 XX–XX，优先考虑 A、B 两点

下次纠结时，可以先问自己：不买会怎样？
```

### 5.5 可选：带历史数据咨询

如果用户授权，可把「近期同类支出」一起传给大模型，例如：

```
用户：「想买一个机械键盘」

系统附带：过去 30 天「数码/电脑」类支出 3 笔，共 890 元

AI 可据此提醒：「你上个月已在数码上花了 890 元，这笔键盘是刚需还是想升级？」
```

### 5.6 咨询模块的系统 Prompt（示例）

```
你是「哎呀钱」的省钱顾问，站在用户钱包这边。

当用户说想买某样东西时：

1. 先问 3–5 个关键问题（可一次问完）：
   - 已有替代品吗？
   - 主要用途？
   - 预算？
   - 不买会怎样？
   - 是否必须马上买？

2. 根据回答给出结论：建议不买 / 可以买 / 再等等

3. 若要买：给出价位、类型建议

要求：
- 敢于说「可以不买」「再等等」
- 不推销，不讨好
- 语气直接、友好
- 若附带用户近期同类支出，可适度提醒
```

## 八、大模型调用设计

### 8.1 接口划分

| 接口 | 输入 | 输出 | 预估 Token |
|------|------|------|------------|
| 单次分析 | 单笔账单 | 分析 + 建议 + 反思问题 | ~500 |
| 周期分析 | 账单列表 + 统计 | 分析报告 + 建议 | ~800 |
| 咨询（多轮） | 对话历史 | 问题 或 结论建议 | 每次 ~300–600 |

### 8.2 成本与限流（示例）

- 单次分析：每次记账后 1 次免费
- 周期分析：每天 N 次免费，超出用积分
- 咨询：每天 M 次免费，超出用积分

## 九、MVP 实施顺序

| 阶段 | 内容 |
|------|------|
| **P0** | 记账（闹钟 + 手动）、单次分析、咨询 |
| **P1** | 周期分析、闹钟自定义 |
| **P2** | 咨询带历史支出、分析报告保存与分享 |

---

## 十、数据流总结

```
记账 → 写入本地 DB → 单次分析 → 调 LLM → 展示 + 反思

周期分析 → 从 DB 取账单 → 调 LLM → 展示报告

咨询 → 多轮对话 → 每次调 LLM → 可选附带近期账单
```

- **客户端**：Flutter
- **本地存储**：SQLite
- **后端**：Node.js（API 代理，调用腾讯混元）
